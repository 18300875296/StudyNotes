###### 模块化
```
模块化的优点:遵守同样的代码规范,降低了沟通成本,方便了各个模块之间的相互调用
```
###### 模块化规范分类
```
社区模块化规范:AMD CMD CommonJS
  由社区提供的这些模块化标准,存在差异性与局限性,并不是浏览器与服务器通用的模块化标准
    AMD CMD 适合用于浏览器端javascript
    CommonJS 适合于服务器端的javascript

官方的模块化规范:ES6模块化规范
```
###### node中的模块化
```
node.js遵循了CommonJS的模块化规范
  导入其他模块使用require()方法
  模块对外共享成员使用module.exports对象
```
###### ES6模块化
```
ES6模块化规范定义:
  每个js都是独立的模块
  导入其他模块成员使用import关键字
  向外共享模块成员使用export关键字
```
###### ES6模块化的基本语法
```
默认导出和默认导入
  默认导出:export default 默认导出的成员
  默认导入:import 接收名称 from '模块标识符'
    注意:每个模块中只能使用一次 export default,否则报错

按需导入和按需导出
  按需导出:
    export let a ='111'
    export let b ={q:'zzzz'}
    export let c =function(){}
  按需导入:
    import {a,b,c} from '模块标识符'   
  注意:
    每个模块可以多次按需导出
    按需导入成员名称和按需导出的名称要保持一致
    按需导入和默认导入可以一起使用
    按需导入时,可以用as关键字重命名: import {a,b as b1,c} from '模块标识符' (b被重命名为b1)

直接导入并执行模块中的代码
  定义:只想单纯地执行某个模块中的代码,并不需要得到模块中向外共享的成员,可以直接导入并执行模块代码
  import '模块标识符'
```
